# 🐇 **しろうさぎアナウンサー紙芝居シリーズ：完全再現・実装仕様書**

## **1. 要件定義書（Requirements Definition）**
本プロジェクトの目的は、難解なIT研修内容を「面白きこともなき世を面白く」 伝えるために、**「認知負荷の極小化」**と**「感情のエンゲージメント」**を両立させた学習体験を提供することです。

*   **ターゲット：** 初学者、新卒エンジニア、およびIT研修講師。
*   **品質目標：** デザインは「感性」ではなく「設計」で行い、品質は「気合」ではなく「システム」で保証します。
*   **再現性要件：** 新規描画をゼロにし、既存のソース（しろうさぎ画像）の切り貼りと色、カメラワークの切り替えだけで「生きた番組」を構成します。

## **2. 詳細設計書（Detailed Design）**

### **① 視覚・構図設計（Visual Layout）**
脳が情報を処理する**0.05秒（50ms）**をハックするための設計を行います。
*   **基本レイアウト：** **「PiP小窓（左：キャラクター）」＋「メイン画面（右：図解）」**の固定。画面が切り替わってもキャラクターが常駐することで、視聴者に「安心感」を与えます。
*   **余白ルール：** 画面の**3/4を余白**（呼吸するための空間）として確保します。これは認知負荷を軽減するための「引き算の美学」です。
*   **視線誘導：** 左上から右下へ流れる**「Z字スキャン」**に合わせ、重要要素を配置します。
*   **9つの構図マトリクス：** 安定には「三分割法」、誘導には「黄金螺旋」、強調には「余白」を使い分けます。

### **② キャラクター設定（Character Bible）**
*   **ビジュアル：** ふわふわの白い毛並み、丸い耳、青い蝶ネクタイ。
*   **パーソナリティ：** 丁寧、やさしい、落ち着いた語り口。
*   **台本ルール：** 語尾は「〜ですね」「〜ですよ」に統一し、専門用語を日常の例え（例：質問の4ステップ）で解説します。

## **3. 実装仕様書：アニメーション演出（Animation & Color）**

### **① タイムラインとカラー心理学**
スライドの役割に応じて、背景の水彩トーンを切り替えます。
1.  **第1幕（導入）：** 青・グレー系（安定・基本セット）。
2.  **第2幕（課題）：** **赤・ピンク系**（注意喚起・キーメッセージ強調）。
3.  **第3幕（解決）：** 再び青系（安心。パン・ズームで図表へ視線誘導）。
4.  **第4幕（行動）：** **オレンジ・暖色系**（キャラクターによる直接的な語りかけ）。
5.  **第5幕（結）：** ホワイトアウト（丁寧な挨拶で終了）。

### **② カメラワークとテロップ**
*   **動き：** 静止画に対して「ズーム」と「パン」のみを使用し、時間経過と視線誘導を付与します。
*   **下部テロップ：** Tipsや要点を常時ループ表示し、情報密度を高めます。

## **4. 脚本ガイドライン（Scripting Framework）**
各スライドの構成は以下の**「導入・説明・転・結」**の4構成を基本とします。
*   **文章量：** 1スライドあたり**120〜150文字**（読み上げ速度の安定化のため）。
*   **メンター・コミュニケーション要素：** ソース資料に基づき、「答えではなく問いを返す」「過去の自分と比較する」「確認は武器、確定は盾」といった具体的な原則を盛り込みます。
