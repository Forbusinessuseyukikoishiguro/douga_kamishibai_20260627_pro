# 【IT研修講師向け】Nano-banana × Easy Banana で「学習4コマ漫画」を10分量産する完全ガイド

:::note info
この記事は **IT研修講師・社内教育担当者** に向けて書いています。
「Pythonの例外処理ってどう教えればいい？」「新人に障害対応フローを覚えてもらいたい」——そういった **教育コンテンツをAI漫画で爆速量産** する手順を、ステップバイステップで解説します。
:::

---

## はじめに：なぜ「漫画」が研修教材として強いのか

テキスト教材は読み飛ばされる。スライドは眠くなる。

でも漫画は——**キャラクターが「失敗」「気づき」「解決」を体験する様子** を視覚的に追えるため、認知負荷が下がり記憶に残りやすいです。

さらに2025年8月末に登場した **Google製画像AI「Nano-banana（ナノバナナ）」** により、**キャラクターの一貫性を保ったまま複数コマを量産** できるようになりました。

これは研修講師にとって革命です。

| 従来の方法 | AI漫画（本記事の手法） |
|---|---|
| イラストレーターに依頼 → 数万円〜 | 素材代のみ（1コマ約20円〜） |
| 納品まで数週間 | 当日完成 |
| 修正依頼がしにくい | プロンプト変更で即再生成 |
| キャラが毎回変わる | キャラシートで一貫性を維持 |

## STEP 0：全体フロー（10分の内訳）

```
[1分] テーマ決め・ネーム（台本）をAIで生成
[2分] キャラクターシートをNano-bananaで生成
[5分] 各コマの静止画を生成（4コマ × 約1〜2回ガチャ）
[2分] Canvaで吹き出し・文字を入れて完成
```

慣れれば4コマ1本が **10分以内** に完成します。複数ページの長編も同じフローの繰り返しです。

## STEP 1：教育テーマからネームを作る（1分）

### 1-1. ChatGPT / Claudeへの依頼プロンプト

以下のテンプレをそのまま使えます。

```
あなたはIT研修漫画のシナリオライターです。
以下のテーマで日本風4コマ漫画のネームを作ってください。

【テーマ】{ここに入れる例：Gitのコンフリクト解消}
【主人公】新入社員エンジニア（研修生・名前はユウ）
【先輩キャラ】頼れる先輩エンジニア（名前はサクラ先輩）

出力形式：
- 1コマ目：状況設定（日常・問題の予兆）
- 2コマ目：問題発生（ミス・困惑・エラー）
- 3コマ目：先輩のアドバイス or 気づき
- 4コマ目：解決・まとめ（読者への学び）

各コマに「セリフ」「表情」「背景」を含めてください。
右から左に読む日本マンガ形式で。
```

### 1-2. 生成例（Gitコンフリクトテーマ）

```
1コマ目：
  背景：オフィス・PCデスク
  ユウ：「やったー！機能追加できた！git pushっと」
  表情：得意げ・笑顔

2コマ目：
  背景：PCの画面（エラー表示）
  ユウ：「え…！？CONFLICT って何！？」
  表情：目が点・大汗マーク

3コマ目：
  背景：隣に来るサクラ先輩
  サクラ：「コンフリクトはね、同じ箇所を
           2人が同時に書き換えたサイン。
           怖くないよ、一行ずつ確認するだけ」
  表情：サクラ：優しく微笑む / ユウ：ハッとした顔

4コマ目：
  背景：解決済みのPCとユウ
  ユウ：「コンフリクト＝チームワークの証拠…！
         これ覚えた！」
  サクラ（小さく）：「成長したね」
  表情：ユウ達成感・サクラ微笑む
```

:::note
**教育的に効果的な4コマの型：**
`日常 → 失敗（エラー） → 気づき（メンター） → 定着（読者への学び）`
これはSBIフィードバックモデル（Situation-Behavior-Impact）と構造的に近く、認知科学的にも記憶されやすい形式です。
:::

## STEP 2：キャラクターシートを作る（2分）

ここが **最重要ステップ** です。キャラシートを一度作れば、以降すべてのコマ生成で使い回せます。

### 2-1. Easy Bananaのセットアップ

1. [Easy Banana配布ページ](https://note.com/konho/n/n5e0138afcee5) からv2.9以降をDL
2. `chrome://extensions` を開く → デベロッパーモードをON
3. 「パッケージ化されていない拡張機能を読み込む」でフォルダを選択
4. [fal.ai](https://fal.ai) でAPIキーを取得・$10チャージ
5. Easy Banana設定画面にAPIキーをペースト

### 2-2. キャラクターシートの生成

Easy Bananaを開き、以下を設定します。

**モデル選択：** `nano-banana-pro edit`
**アスペクト比：** `16:9`（コマ割りに使いやすい）

**プロンプト：**
```
新入社員の日本人女性エンジニア。
黒髪ショートカット。白いシャツにネクタイ。
三面図のキャラクターシート（正面・横・斜め）。
表情バリエーション付き（笑顔・困惑・驚き・得意げ）。
アニメ・漫画スタイル。文字は除去。
```

:::note warn
**研修教材向けキャラデザの注意点：**
- 左右非対称なデザイン（片目だけメガネ等）は避ける → コマごとにズレる
- 複雑な柄の服は避ける → コマごとにデザインが変わる
- シンプルな特徴（黒髪×ネクタイ等）を「アイデンティティ」にする
:::

**生成例：**
シートが出たら保存。このファイルが **以降すべてのコマ生成のベース** になります。

## STEP 3：各コマの静止画を生成する（5分）

4コマそれぞれを生成します。1コマあたり30秒〜1分が目安です。

### 3-1. 基本プロンプト構造

```
[構図指定] [場所・背景] [キャラの動作・表情] [漫画的演出]
```

### 3-2. コマ別プロンプト例（Gitコンフリクトテーマ）

#### 1コマ目：日常・得意げな場面

```
モデル：nano-banana-pro edit
参考画像：キャラクターシート

プロンプト：
オフィスのデスク。PCに向かって指を突き出す女性エンジニア。
得意げな笑顔。顔のズームイン。3/4のアングル。
漫画のような表現。白い背景に星マーク。
```

#### 2コマ目：問題発生・困惑

```
PCの画面に赤いエラーテキスト。
目が点になって大汗マークの女性エンジニア。
顔の極端なズームイン。ローアングル。
漫画のような大きな汗マーク。集中線。3/4のアングル。
```

#### 3コマ目：先輩アドバイス（2キャラ登場）

:::note
**2キャラを1コマに入れる場合は「肩越しショット」が有効です：**
:::

```
モデル：nano-banana-pro edit
参考画像：[キャラシート1（ユウ）] + [キャラシート2（サクラ先輩）]

プロンプト：
オフィス。先輩エンジニア越しに、新入社員を撮影。
先輩は後ろ姿・上半身。新入社員はハッとした顔。
先輩にはボケ処理。ハイアングル。顔のズームイン。
```

#### 4コマ目：解決・達成感

```
デスクに戻った女性エンジニア。
両手をグッと握りしめ、目から炎が出ている。
やる気に満ちた表情。漫画のような表現。燃える背景。
顔のズームイン。ローアングル。
```

### 3-3. うまく出ないときの対処法

| 問題 | 対処 |
|---|---|
| 全身が写って小さい | 「顔の極端なズームイン」を追加 |
| 真正面で単調 | 「3/4のアングル」を追加 |
| 漫画っぽくない | 「漫画のような表現」を先頭に |
| 2キャラが混ざる | 「肩越しショット」+「ボケ処理」で分離 |
| 服のデザインがズレる | キャラシートを再読み込みして再生成 |

## STEP 4：Canvaで吹き出し・コマ割りを作る（2分）

### 4-1. コマ割りテンプレートの使い方

Canvaで「漫画 4コマ」で検索すると無料テンプレートがあります。なければ以下で作成します。

```
キャンバスサイズ：600 × 1600px（縦型・スマホ最適）
コマ割り：縦4分割（各コマ 600 × 370px）
コマ間の余白：10px（黒）
```

### 4-2. 吹き出しの追加

1. 「素材」→「図形」→「吹き出し」を選択
2. 生成した静止画をコマにドラッグ
3. ネームのセリフをそのまま入力

**フォント推奨：**
- 本文：**源ノ角ゴシック JP（Bold）** / **Noto Sans JP**
- 効果音・強調：**ロゴたいぷゴシック** / **こども丸ゴシック**

### 4-3. 誤字・デザインのズレはCanvaで修正

Nano-bananaが誤字を出した場合（看板の文字など）：
- Canvaの「背景除去」→上に正しいテキストを重ねる
- または Canvaの「消しゴム」で該当部分を消して再上書き

## STEP 5：複数ページの長編漫画を作る

4コマを複数枚つなげると、**1エピソード完結型の研修漫画** が作れます。

### 5-1. 推奨ページ構成（エピソード1本）

```
P1（4コマ）：問題提起・エラー発生
P2（4コマ）：原因の調査・先輩への相談
P3（4コマ）：解決プロセス・手順の説明
P4（4コマ）：解決・学びの定着・次回予告
```

合計16コマ = 約40分（慣れれば）

### 5-2. キャラクターの一貫性を保つコツ

**「壁ドン」絵柄統一テクニック（上級者向け）：**

複数キャラの絵柄がバラバラになる場合：
1. 絵柄を合わせたいキャラ2人を同じコマに出す（壁ドンや対話シーン）
2. そのコマの画像を次の生成の参考画像に入れる
3. 絵柄が混ざり合い、自然に統一される

### 5-3. 研修テーマ別プロンプト集

以下、コピペで使えるシーン別プロンプト集です。

#### 🖥️ エラー・デバッグ系

```
# 困惑シーン（汎用）
PCのエラー画面。漫画のように激しく困惑している女性エンジニア。
口から魂が出ている。白い背景。集中線。顔のズームイン。ローアングル。

# 解決の瞬間
パソコンの前で「わかった！」と叫ぶ女性。
漫画のように激しく喜んでいる。背景はパステルカラーの星。
顔のズームイン。手を掲げている。
```

#### 👥 コードレビュー・チームワーク系

```
# レビューを受けるシーン（2キャラ）
会議室。ホワイトボード前のシニアエンジニア越しに、若手を撮影。
若手はメモを取りながら真剣な表情。シニアにボケ処理。ハイアングル。

# ペアプロシーン
横並びでPCに向かう2人のエンジニア。
会話しながら画面を指差している。ローアングル。3/4のアングル。
```

#### 📋 ドキュメント・仕様書系

```
# 読めない仕様書に絶望
積み上がった書類の山。その前で頭を抱えてうずくまる女性エンジニア。
どんよりした雲の背景。漫画のような落ち込み表現。顔のズームイン。

# ドキュメント整備の達成感
整理されたフォルダ構成が画面に映るPC。
拳を突き上げて喜ぶ女性。漫画的な輝き。3/4のアングル。
```

#### 🔐 セキュリティ研修系

```
# 不審メールを発見（緊張感）
暗いオフィス。PCの画面に怪しいメールが表示されている。
画面を凝視する女性エンジニア。目のズームイン。ブルーモーメント。

# 上司への報告
上司の席へ向かう女性エンジニア。緊張した表情。
廊下の肩越しショット。ハイアングル。
```

## STEP 6：note.com / Zenn / Qiitaへの展開

作成した漫画は複数プラットフォームで転用できます。

### 媒体別の活用方法

| 媒体 | 向いているコンテンツ | 備考 |
|---|---|---|
| **Qiita** | 技術的な手順解説4コマ | コードブロックと組み合わせる |
| **Zenn** | 設計思想・考え方を伝える漫画 | Book機能でエピソード連載 |
| **note.com** | 研修文化・チームあるあるネタ | 感情に訴えるコンテンツ向き |
| **社内Confluence / Notion** | オンボーディング教材 | ページ内に画像として埋め込む |
| **研修スライド** | 冒頭つかみ・休憩明けの掴み | PPTXに画像挿入 |

### Qiita記事での埋め込み方法

```markdown
![Gitコンフリクト解説漫画P1](./images/git_conflict_p1.png)

> **📖 解説：** 上のコマで描いたように、コンフリクトは...
```

画像直後に文字解説を入れると、**漫画で掴んで文字で深める** という二段構えの教材になります。

## コスト試算：月50本の漫画を量産する場合

```
Nano-banana（プロ）：1コマ約20円
4コマ漫画 1本：4コマ × 20円 × ガチャ平均2回 = 160円
月50本：160円 × 50本 = 8,000円

ChatGPT（ネーム生成）：月額 $20 ≒ 3,000円
Canva Pro（テンプレ）：月額 1,500円

合計：約12,500円/月
```

これで月50本の教材漫画が量産できます。イラストレーターに依頼する場合の **100分の1以下のコスト** です。

```
✅ テーマ決め → ChatGPTで4コマネームを30秒生成
✅ キャラシート → Nano-banana-pro editで一度作れば使い回し可

```
